# Этажи Директ — управление рекламой недвижимости

Специализированный скилл для агентства недвижимости «Этажи». Оркестрирует MCP-инструменты Яндекс Директа, Метрики и Wordstat для создания, анализа и оптимизации рекламных кампаний.

**Компания:** Этажи — крупнейшее агентство недвижимости России. 70+ городов, домен etagi.com.
**MCP-сервер:** `leadgen-mcp` (Яндекс Директ + Метрика + Wordstat).
**Аккаунт Директа:** `etagi click` (default) — агентский, 76 клиентов-городов.
**Аккаунт Метрики:** `etagi.agent` — 467 счётчиков, 70 активных городов.

### Направление: покупатель vs продавец

- **Покупатель** (95% кампаний) — человек ищет объект для покупки/аренды. Посадочная — каталог с фильтрами.
- **Продавец** (5%) — человек хочет разместить свой объект через Этажи. Посадочная — форма оценки/размещения.
- Это РАЗНЫЕ кампании с разными ключевыми, текстами и посадочными. Не смешивать.

## Перед любой работой

### Определить город и конфигурацию

1. **Город** — спроси если не указан
2. **Конфигурация города** — вызови `get_city_config(city="<город>")` → вернёт client_login, counter_id, domain, geo_region_id
3. **Цели и ценности** — вызови `get_conversion_values(client_login=<login>, counter_id=<id>)` → goal_id обеих целей + рекомендуемые value + priority_goals_json

### Принцип рассуждения

**Ты НЕ копировальный аппарат.** При каждом действии:
- Загрузи данные по аналогичным кампаниям (тот же тип в других городах)
- Проанализируй что работает (CTR, конверсии) и что нет
- Предложи улучшения с конкретным обоснованием
- Для каждого нового заголовка/текста — объясни гипотезу

### Шаг 1. Определение параметров

Спроси у пользователя (если не указал):

1. **Город** — какой город запускаем?
2. **Тематика** — что рекламируем?
3. **Тип размещения** — выбор:

| Тип | API | Когда использовать |
|---|---|---|
| Поиск | TEXT_CAMPAIGN (network=SERVING_OFF) | Горячий спрос, высокий интент |
| РСЯ | TEXT_CAMPAIGN (search strategy=SERVING_OFF) | Охват, ретаргетинг, имидж |
| ЕПК / Мастер | UNIFIED_CAMPAIGN (API v501) | Автоматизированная, для тестов |
| Динамические | DYNAMIC_TEXT_CAMPAIGN | Большая база объектов, фид |
| Медийная | CPM_BANNER_CAMPAIGN | Охват, узнаваемость, CPM |

4. **Бюджет** — недельный, в рублях
5. **Посадочная** — [site] (etagi.com) или [LP] (etagi.net)

### Шаг 2. Подготовка конфигурации

```
1. Вызови get_city_config(city="<город>") → counter_id, client_login, domain, geo_region_id
2. Вызови get_conversion_values(client_login=<login>, counter_id=<id>)
   → Вернёт: goal_id обеих целей + рекомендуемые value + готовый priority_goals JSON
   → Сохрани priority_goals_json для Шага 6
3. Загрузи config/campaign_naming.md → конвенция именования кампании
4. Вызови get_utm_config(city="<город>", theme="<тематика>", placement="<тип>")
   → Вернёт готовый tracking_params для Шага 6
```

### Шаг 3. Анализ аналогов

```
1. Найди аналогичные кампании в кабинете:

2. Получи статистику лучшей аналогичной:

     field_names="CampaignName,Impressions,Clicks,Cost,Ctr,Conversions,CostPerConversion",
     goal_ids=<goal_id_донора>, attribution="LYDC")

3. Получи объявления донора:
   get_ads(client_login=<город-донор>, campaign_ids=<id>,
     text_ad_field_names="Title,Title2,Text,SitelinkSetId,AdExtensions")

4. Напиши секцию "Анализ и гипотезы":
   - Что работает у аналога (конкретные цифры)
   - Что можно улучшить и почему
   - Новые идеи с обоснованием
```

### Шаг 4. Сбор семантики (многораундовое расширение)

**Цель:** 30–80 ключевых фраз для типичной кампании. 14 фраз — слишком мало, 200+ — переспам. Ядро из get_semantic_cluster — лишь отправная точка.

#### Раунд 1: Базовая матрица (seeds)

```
1. Вызови get_semantic_cluster(theme="<тематика>")
   → modifiers, objects, extra_keys, negative_keywords

2. Построй матрицу: {модификатор} × {объект} + {город}
   Пример (вторичка, Омск):
     "купить квартиру омск", "купить однушку омск", "купить двушку омск",
     "квартира омск цена", "однокомнатная омск вторичка", ...
   → ~10-20 seed-фраз (не больше, это входные данные для расширения)

3. Если есть extra_keys в кластере — добавь их с подстановкой города
```

#### Раунд 2: Расширение через Wordstat (КЛЮЧЕВОЙ ЭТАП)

```
1. Вызови check_search_volume(
     phrases="<все seed-фразы через запятую>",
     region_ids="<geo_region_id города>"
   )
   → API вернёт для каждой фразы блок SearchedWith — реальные запросы пользователей

2. Из SearchedWith собери ВСЕ фразы с Shows >= 10 (для малых городов >= 5)

3. Какие фразы ценны в SearchedWith:
   - Гео-уточнения: район, микрорайон, улица ("квартира кировский район омск")
   - Ценовые: "до 2 млн", "недорого", "дешевые"
   - Типовые уточнения: "с балконом", "с ремонтом", "от собственника"
   - Синонимы: "однушка" ↔ "1 комнатная" ↔ "однокомнатная"
   - Long-tail: "купить однокомнатную квартиру омск вторичное жилье"

4. Отфильтруй нерелевантные — сравни с negative_keywords из кластера:
   - Аренда/сдача: снять, сдам, аренда, посуточно
   - Информационные: как, почему, отзывы, рейтинг, форум
   - Чужие бренды: авито, циан, домклик (если не в наших ключевых)
   - Другая тематика: новостройка, застройщик, жк (для вторички)
   - Нецелевые: ремонт, дизайн, курсовая, реферат
```

#### Раунд 3: Второй уровень расширения (опционально)

```
Если в Раунде 2 обнаружены высокочастотные фразы (Shows > 100),
которых НЕ было в seeds — запусти ещё один check_search_volume по ним.

Пример: seed "купить квартиру омск" → SearchedWith показал
"квартира левый берег омск" (Shows 320) — это новый кластер,
у которого свои вложенные запросы.

Лимит: не более 2 раундов расширения (seeds → round2 → round3).
Больше — diminishing returns.
```

#### Раунд 4: Финализация и подготовка к кластеризации

```
1. Объедини все собранные фразы, убери дубли (регистр, порядок слов)
2. Отсортируй по Shows (частотности) — от большей к меньшей
3. Проверь каждую на релевантность:
   - Соответствует тематике кампании?
   - Покупательский интент (не информационный)?
   - Подходит к посадочной странице?

4. Собери итоговый список минус-слов:
   - negative_keywords из get_semantic_cluster
   - + новые нерелевантные паттерны, обнаруженные в SearchedWith
   - Минус-слова идут на уровень кампании (negative_keywords в add_campaign)

5. Результат → массив фраз [{phrase, shows}] для Шага 5 (кластеризация)
```

**Важно:** check_search_volume принимает до 10 фраз за раз (лимит API). Разбивай на батчи по 10. Всегда передавай region_ids города — без гео данные будут по всей России и нерелевантны.

### Шаг 5. Кластеризация семантики → структура групп объявлений

**Ключевой шаг.** Собранную семантику нельзя складывать в одну группу — нужно разбить на кластеры, каждый из которых станет отдельной группой объявлений с индивидуальным контентом.

#### Принципы кластеризации

```
Агент САМОСТОЯТЕЛЬНО решает, как разбить семантику. Ориентиры:

1. ГЕО-КЛАСТЕРЫ (приоритет если обнаружены)
   Если в SearchedWith найдены фразы с районами/микрорайонами — это отдельные кластеры.
   Каждый район → своя группа с:
   - Названием района в заголовке объявления
   - Фильтром района в посадочной URL (если сайт поддерживает, см. references/site_filters.md)
   - Ключевыми только по этому району

   Пример (Омск, вторичка, однокомнатные):
   | Кластер | Ключевые (примеры) | Заголовок | Посадочная |
   |---------|-------------------|-----------|------------|
   | Нефтяники | "квартира нефтяники омск", "однушка нефтяники" | Однушки в Нефтяниках | /realty/?rooms=1&district=... |
   | Кировский | "квартира кировский район омск" | Однушки в Кировском | /realty/?rooms=1&district=... |
   | Левый берег | "квартира левый берег омск" | Однушки на Левом берегу | /realty/?rooms=1&district=... |

2. ИНТЕНТ-КЛАСТЕРЫ
   - «Купить» (прямой интент) — "купить квартиру омск"
   - «Вторичка/от собственника» (уточнённый) — "вторичка омск", "от собственника"
   - «Цена/недорого» (ценовой) — "квартира омск недорого", "дешевые квартиры"

3. ОБЪЕКТНЫЙ КЛАСТЕР (если кампания шире одного типа)
   - Однушки, двушки, студии — если все в одной кампании

4. ОБЩИЙ кластер (catch-all)
   Широкие фразы без уточнений: "квартира омск", "однокомнатная омск"
   → Общий заголовок без гео-уточнения, базовая посадочная
```

#### Алгоритм кластеризации

```
1. СНАЧАЛА извлеки гео-кластеры:
   - Пройди по всем фразам, найди упоминания районов/микрорайонов
   - Группируй: все фразы с "нефтяники" → кластер "Нефтяники"
   - Минимум для гео-кластера: 3+ фразы ИЛИ 1 фраза с Shows >= 50

2. ЗАТЕМ остаток разбей по интенту:
   - Фразы с "купить/покупка" → прямой интент
   - Фразы с "цена/недорого/дешев" → ценовой
   - Фразы с "вторичка/собственник/хозяин/без посредников" → условия
   - Остальное → общий кластер

3. ОЦЕНИ размер кластеров:
   - Если кластер < 3 фраз и Shows < 50 — влей в ближайший по смыслу
   - Если кластер > 40 фраз — подумай, можно ли разбить ещё
   - Оптимально: 5-20 фраз на кластер

4. ДЛЯ КАЖДОГО кластера определи:
   - Название группы (для add_adgroup): "{Тематика} — {кластер}"
   - Ключевые фразы (для add_keywords)
   - Посадочный URL: базовый + фильтры (если есть в site_filters.md)
   - Специфика заголовка: что добавить/изменить в Title
```

```
4. Загрузи library/extensions.md → уточнения

ДЛЯ КАЖДОГО КЛАСТЕРА создай индивидуальный контент:

8. Быстрые ссылки:
   - Общие для всех кластеров ИЛИ адаптированные (если есть смысл)

9. Юридическая проверка → LEGAL.md (все тексты всех кластеров)

Для каждого НОВОГО заголовка/текста — объясни гипотезу.
Проверь лимиты символов (Title: 56, Title2: 30, Text: 81).
```

### Шаг 7. Создание через API (мульти-группы)

```
1. КАМПАНИЯ (одна на все кластеры):
   add_campaign(
     client_login=<login>,
     name="<Город> | <Тип> | <Тематика> | <Детали> | [<посадка>]",
     daily_budget_amount=<бюджет в рублях>,
     search_strategy="WB_MAXIMUM_CONVERSION_RATE",
     network_strategy="SERVING_OFF",
     priority_goals=<priority_goals_json из get_conversion_values>,
     counter_ids="<counter_id>",
     negative_keywords="<минус-слова через запятую>",
     start_date="<YYYY-MM-DD>",
     settings='[{"option":"ENABLE_AREA_OF_INTEREST_TARGETING","value":false}]'
   )

2. ЦИКЛ ПО КЛАСТЕРАМ — для каждого кластера:

   a) add_adgroup(
        client_login=<login>,
        campaign_id=<campaign_id>,
        name="<Тематика> — <Название кластера>",
        region_ids=[<регион>]
      )

   b) add_keywords(
        client_login=<login>,
        adgroup_id=<adgroup_id>,
        keywords="<ключевые ЭТОГО кластера через запятую>"
      )
      // ОБЯЗАТЕЛЬНЫЙ ШАГ — без ключевых группа работает только на автотаргетинге!

   c) update_adgroup(adgroup_id=<id>,
        tracking_params=<tracking_params из get_utm_config>
      )

   d) add_ad × 2 (два варианта для A/B теста)
      // Заголовки и тексты — индивидуальные для кластера!
      // Гео-кластер → район в Title
      → Запомни banner_ids этой группы

   e) add_sitelinks + привязать к объявлениям

   f) add_ad_extension (уточнения) + привязать

3. Собери ВСЕ banner_ids из всех кластеров → для Шага 8 (метки)
```

> **Именование групп:** `"Вторичка Однушки — Общий"`, `"Вторичка Однушки — Нефтяники"`, `"Вторичка Однушки — Кировский"` и т.д.
> **Бюджет:** один на кампанию, Директ сам распределяет между группами по эффективности.

### Шаг 8. Метки (ОБЯЗАТЕЛЬНО)

```
1. Загрузи config/labels.md → определи метки по тематике и направлению
2. Собери ВСЕ banner_ids из Шага 7 (из всех кластеров/групп)
3. Вызови add_labels(
     client_login=<login>,
     campaign_id=<campaign_id>,
     banner_ids="<все banner_ids через запятую>",
     labels="Лидген,Вторичка,Покупатель"
   )
   → Автоматически создаст метки в каталоге кампании + назначит на объявления
4. Проверь через get_labels(client_login=<login>, banner_ids="<id1>,<id2>")
```

> **Без меток статистика не попадёт в отчёты.** Не пропускай этот шаг.
> Метки назначаются на объявления (= группу в интерфейсе) через API v4 Live.
> Нужен campaign_id (каталог) + banner_ids (назначение).
> Одни и те же метки — на ВСЕ объявления всех кластеров кампании.

### Шаг 9. Документирование

Дополнительно зафиксируй:

## ВЕТКА 2: АНАЛИЗ кампаний

### Источники данных

| Источник | Инструмент | Когда |
|---|---|---|
| **Яндекс Директ API** | `get_campaign_stats` | Расход, клики, показы, CTR, конверсии |
| **Яндекс Метрика API** | `metrika_get_direct_report` | Визиты, отказы, глубина, конверсии |
| **Внешний дашборд** | _(будущий MCP #2)_ | Агрегированные данные, быстрая сводка |

### Flow

```
1. ОПРЕДЕЛИТЬ СКОУП
   - Одна кампания (по ID или имени) → детальный аудит
   - Все кампании города → дашборд
   - Все города → сводка

2. СБОР СТАТИСТИКИ
   get_campaign_stats(
     client_login=<login>,
     campaign_id=<id>,  // или без → все кампании
     date_from=<7/30 дней назад>,
     date_to=<вчера>,
     field_names="CampaignName,Impressions,Clicks,Cost,Ctr,AvgCpc,Conversions,CostPerConversion",
     goal_ids=<goal_id>,
     attribution="LYDC"
   )

3. ДАННЫЕ МЕТРИКИ (для детального аудита)
   metrika_get_direct_report(
     counter_id=<id>,
     account="etagi.agent",
     date1=<7 дней назад>,
     date2=<вчера>,
     utm_campaign=<campaign_id>
   )

4. ОЦЕНКА ЗДОРОВЬЯ
   | Метрика | GOOD | ATTENTION | CRITICAL |
   |---------|------|-----------|----------|
   | CPA | < порог из benchmarks | < порог × 1.5 | > порог × 1.5 |
   | CTR (поиск) | > 5% | 3-5% | < 3% |
   | CTR (РСЯ) | > 0.5% | 0.3-0.5% | < 0.3% |
   | Отказы | < 30% | 30-50% | > 50% |

5. ДИАГНОСТИКА ПРОБЛЕМНЫХ
   - Поисковые запросы: get_search_queries → waste / opportunities
   - CTR объявлений: get_ad_stats → какое проигрывает
   - Ключевые: get_criteria_stats → неэффективные фразы

6. ОТЧЁТ
   - Сводная таблица с светофором
   - Конкретные рекомендации (входные данные для ОПТИМИЗАЦИИ)
   - Обновление campaigns/<utm>.md
```

## ВЕТКА 3: ОПТИМИЗАЦИЯ

**Принцип:** ВСЕГДА начинай с данных из ветки АНАЛИЗ. Не оптимизируй вслепую.

### Подветки (выбор по ситуации)

#### 3.1 Минус-слова (высокий CPA, мусорные запросы)

```
1. get_search_queries(campaign_ids=<id>, date_from=<30 дней>, attribution="LYDC")
2. Анализ: расход > 0, конверсии = 0 → кандидаты в минус
3. Информационные запросы → в минус (сверить с минус-словами из get_semantic_cluster)
4. ПОДТВЕРЖДЕНИЕ пользователя перед применением
5. update_campaign(campaign_id, negative_keywords=<список>)
```

#### 3.2 Новые ключевые (упущенный спрос)

```
1. get_search_queries → запросы с конверсиями, но не в ключевых
2. check_search_volume(phrases="<найденные фразы>", region_ids=<гео>)
   → Проверить частотность + собрать SearchedWith для расширения
3. Отфильтровать нерелевантные (см. Шаг 4, Раунд 2 — логика фильтрации)
4. ПОДТВЕРЖДЕНИЕ пользователя (показать таблицу: фраза + Shows)
5. add_keywords(adgroup_id=<id>, keywords="<фразы через запятую>")
```

#### 3.3 A/B тест объявлений (низкий CTR)

```
1. get_ad_stats → определить слабое объявление
2. Загрузить library/titles.md, library/texts.md
3. Создать новый вариант с гипотезой
4. ПОДТВЕРЖДЕНИЕ пользователя
5. add_ad(adgroup_id, ...)
6. Через 2-3 недели: сравнить, остановить проигравшего
```

#### 3.4 Смена стратегии (достаточно конверсий)

```
Условие: 10+ конверсий/нед стабильно 2-3 недели подряд
1. ПОДТВЕРЖДЕНИЕ пользователя (лучше в чт-пт)
2. update_campaign(search_strategy="AVERAGE_CPA", average_cpa=<целевой CPA>)
```

#### 3.5 Бюджет

```
- Увеличение: не более +30% за раз, после 2+ недель стабильного CPA
- Снижение: если CPA выше нормы и оптимизация не помогла
- Сезонность: wordstat_dynamics → скорректировать по сезону
- ПОДТВЕРЖДЕНИЕ пользователя для любых изменений > 10%
```

---

## Правила безопасности

1. **Изменения стратегии/цели/бюджета >30%** — ТОЛЬКО с подтверждением пользователя
2. **Лучше в чт-пт** — изменения вызывают «разведку» алгоритма
3. **Не удалять объявления** — только update_ad для изменений
4. **Атрибуция** — всегда LYDC в отчётах
5. **Бюджет** — в РУБЛЯХ, недельный, НЕ микроюниты
6. **Юридическая проверка** — все тексты через LEGAL.md

## Источники данных: MCP-инструменты vs файлы скилла

### MCP-инструменты (вызываются через сервер, НЕ загружают контекст)

| Инструмент | Заменяет файл | Что возвращает |
|---|---|---|
| `get_city_config(city)` | `config/counters.md`, `config/utm_reference.md` | counter_id, client_login, domain, geo_region_id |
| `get_utm_config(city, theme, placement)` | `config/utm_reference.md` + ручная сборка | Готовый tracking_params с utm_content |
| `get_conversion_values(client_login, counter_id)` | `config/goals.md`, `references/city_benchmarks.md` | goal_id, value, priority_goals_json |
| `get_sitelink_templates(theme, city)` | `library/sitelinks.md` | Наборы быстрых ссылок с доменом города |
| `get_semantic_cluster(theme)` | `library/semantic_clusters.md`, `library/negative_keywords.md` | Модификаторы, объекты, минус-слова |
| `get_blocked_placements()` | `config/blocked_placements.md` | 400+ заблокированных площадок РСЯ |

### Файлы скилла (загружаются в контекст при необходимости)

| Файл | Описание | Когда загружать |
|---|---|---|
| `config/campaign_naming.md` | Конвенция именования кампаний | При создании кампании |
| `config/labels.md` | Метки кампаний для аналитики | При создании кампании (ОБЯЗАТЕЛЬНО) |
| `library/titles.md` | Заголовки объявлений | При создании / A/B тесте |
| `library/texts.md` | Тексты объявлений | При создании / A/B тесте |
| `library/extensions.md` | Уточнения (callouts) | При создании кампании |
| `references/site_structure.md` | URL-дерево посадочных etagi.com | При выборе посадочной |
| `references/site_filters.md` | Фильтры сайта через URL-параметры | При создании посадочных для гео-кластеров |

> **Правило:** сначала используй MCP-инструменты (не тратят контекст). Файлы загружай только если нужных данных нет в MCP.
