# Мониторинг позиций в Яндексе (SERP Monitor)

Скилл для мониторинга рекламных и органических позиций в поисковой выдаче Яндекса через **Yandex Search API v2 (Yandex Cloud)**.

**Зачем:** Позиции в выдаче напрямую влияют на CTR и стоимость клика. Падение позиций = рост CPA. Мониторинг позволяет вовремя реагировать: скорректировать ставки, обновить объявления, пересмотреть стратегию.

**API:** `https://searchapi.api.cloud.yandex.net/v2/web/search` · auth через `Authorization: Api-Key` сервисного аккаунта. Настройка — см. [`config/README.md`](config/README.md).

## Доступные скрипты

### search.sh — поиск в Яндексе (основной)
```bash
scripts/search.sh --query "купить квартиру омск"
scripts/search.sh --query "купить квартиру" --region 66          # Нижний Новгород
scripts/search.sh --query "купить квартиру" --domain etagi.com   # свой домен
```
Выдаёт таблицу top-20 с доменами, заголовками, позицией своего домена (★). Кеш 6ч.

**Важно:** В выдаче позиции 1-4 на коммерческих запросах обычно рекламные (premium), 5-8 — верх органики/гарантия. Это не всегда видно напрямую, учитывай при анализе.

### track_position.sh — отслеживание динамики
```bash
scripts/track_position.sh --query "купить квартиру омск"
scripts/track_position.sh --query "купить квартиру" --region 66
```
Проверяет текущую позицию и сохраняет в CSV-историю (`data/serp-history/`). При 7+ замерах показывает тренд.

### search_region.sh — справочник регионов
```bash
scripts/search_region.sh                    # все регионы
scripts/search_region.sh --search "Нижний"  # поиск по имени
```

## Workflow: анализ конкурентов

Анализ конкурентов не требует отдельного скрипта — Claude анализирует выдачу из `search.sh`.

### Шаги:

1. **Собрать выдачу** — запустить `search.sh` для 5-10 ключевых запросов кампании
2. **Claude анализирует рекламные тексты** конкурентов из результатов:
   - Какие УТП используют (цена, ипотека, рассрочка, опыт, количество объектов)
   - Какие CTA в заголовках («Подобрать», «Рассчитать ипотеку», «Смотреть каталог»)
   - Какие домены стабильно в топ-4 (= активно рекламируются)
3. **Рекомендации по объявлениям:**
   - «60% конкурентов указывают цену в заголовке — протестируйте формат "от X млн"»
   - «3 из 5 пишут про ипотеку — добавьте УТП ипотеки в Title2»
   - «Никто не использует "без комиссии" — возможность выделиться»

### Пример запроса пользователя:
> «Проанализируй конкурентов по запросу купить квартиру омск»

Действия:
1. `scripts/search.sh --query "купить квартиру омск" --region 66`
2. Проанализировать заголовки и сниппеты конкурентов в топ-8
3. Выделить паттерны в УТП и CTA
4. Сравнить с текущими объявлениями кампании (MCP `get_ads`)
5. Дать конкретные рекомендации по улучшению текстов → записать в C3 «Анализ аналогов» или в предложение O3.3 «A/B объявлений»

## Интеграция с MCP `yandex-direct`

Для расхода и статистики кампаний используй MCP `yandex-direct`:
- `get_campaign_stats` — расход, клики, показы, конверсии (attribution: LYDC)
- `get_campaigns` / `update_campaign` — проверка и изменение бюджета
- `get_ads` — текущие тексты объявлений для сравнения с конкурентами
- `metrika_get_direct_report` — визиты и отказы из Метрики

## Кросс-ссылки и интеграция

| Ситуация | Действие |
|----------|----------|
| Высокий CPA в Метрике | Проверить позиции — возможно, упали и CTR снизился |
| Позиция упала с #2 на #5 | Проверить бюджет через `get_campaign_stats` — возможно, конкуренты подняли ставки |
| Конкуренты в топ-4, мы на #7 | Рассмотреть повышение ставок или смену стратегии |
| Позиция стабильна, но CPA растёт | Проблема не в позиции — проверить лендинг, отказы в Метрике |
| Новая кампания запущена | Через 3-5 дней начать отслеживание позиций |
| A/B объявлений (O3.3) | До создания новых Title — посмотреть формулировки конкурентов |

### В каких шагах leadgen-флоу используется:

| Шаг | Зачем | Скрипт |
|---|---|---|
| **C3** «Анализ аналогов» | До создания кампании — собрать выдачу по топ-5 запросам тематики, извлечь УТП/CTA конкурентов для гипотез заголовков | `search.sh` (×5 запросов) |
| **A4** «Оценка здоровья» | Если CPA выше порога — проверить не упала ли позиция (корреляция CPA↔позиция) | `track_position.sh` |
| **A5** «Диагностика» | Root-cause: поисковые запросы + позиция на каждом = видно где просели показы | `track_position.sh` |
| **post-apply-monitoring** | T+3 дня после apply — замерить позиции, сверить с прогнозом | `track_position.sh` |
| **O3.3** «A/B объявлений» | До генерации нового Title — посмотреть актуальные формулировки в SERP | `search.sh` |

### Связка с другими скиллами:

- **yandex-metrika** → если позиция упала, проверить CPA и конверсии через `metrika_get_direct_report`
- **yandex-direct** → управление кампаниями, ставками, объявлениями
- **demand-research** → если конкурентов стало больше, исследовать новые ниши через анализ спроса

## Обновление файлов кампаний

После проверки позиций обнови файл `campaigns/<utm>.md` — секция `### Позиции в выдаче`:

```markdown
### Позиции в выдаче
| Дата | Запрос | Регион | Позиция | Тренд | Зона |
|------|--------|--------|---------|-------|------|
| 2026-04-23 | купить квартиру омск | 66 | #3 | стабильно | Premium |
| 2026-04-23 | двухкомнатная квартира омск | 66 | #5 | ↓ с #3 | верх |
```

Обнови `### Где сейчас` если позиция значительно изменилась (падение >3 позиций, выход из Premium-зоны).

## Рекомендуемая частота проверок

- **Еженедельно** (понедельник) — проверить позиции по основным 5-10 запросам каждой кампании
- **После изменений** — через 2-3 дня после смены ставок/стратегии/текстов (post-apply-monitoring)
- **При росте CPA** — немедленно проверить, не упали ли позиции (триггер из A4)
- **Ежемесячно** — полный анализ конкурентов по всем ключевым запросам

## Настройка

- **`YANDEX_API_KEY`** и **`YANDEX_CLOUD_FOLDER_ID`** — из `tokens.env` (см. [`config/README.md`](config/README.md))
- **`SITE_DOMAIN`** — из `tokens.env`, можно переопределить через `--domain`
- **Квота** — платный API, промо-грант нового аккаунта Yandex Cloud ~4000 ₽ / 60 дней. Лимиты каталога смотри в Cloud Console → AI Studio → Search API
