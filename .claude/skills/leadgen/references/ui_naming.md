# Маппинг API-имён Яндекс Директа в UI-названия

Справочник для перевода API-констант (как они видны в коде/запросах MCP) в человеческие имена, как их видит рекламодатель в кабинете Яндекс Директа.

> **Правило использования** (см. `skill.md` → «Коммуникация с пользователем»):
> - В чате, отчётах, превью, `campaigns/<slug>.md` — пиши **UI-имена**.
> - В параметрах вызовов MCP-инструментов (JSON, строки стратегий) — оставляй API-имена (иначе API сломается).
> - Если UI-имя неизвестно — так и скажи («API-константа, точное UI-имя не подтверждено»), не выдумывай.

**Источники:**
- [Яндекс Директ API — Стратегии показов](https://yandex.ru/dev/direct/doc/ru/objects/campaign-strategies)
- [Справка Яндекс Директ — Выбор стратегии](https://yandex.ru/support/direct/ru/strategies/select-strategy)

## Стратегии показа (`search_strategy` / `network_strategy`)

| API | UI (в кабинете Директа) |
|---|---|
| `WB_MAXIMUM_CONVERSION_RATE` | **«Максимум конверсий»** (автобюджет, оплата за клики). Требует цели с ценностью, недельный бюджет. |
| `AVERAGE_CPA` | **«Оптимизация конверсий»** с удержанием средней цены конверсии (оплата за клики). |
| `AVERAGE_CPA_MULTIPLE_GOALS` | «Максимум конверсий» с оплатой за клики и несколькими целями. |
| `PAY_FOR_CONVERSION` | **«Оплата за конверсии»** (оптимизация конверсий). |
| `PAY_FOR_CONVERSION_CRR` | «Оптимизация доли рекламных расходов» с оплатой за конверсии. |
| `WB_MAXIMUM_CLICKS` | **«Максимум кликов»** (автобюджет). |
| `AVERAGE_CPC` | «Оптимизация кликов» с ограничением средней цены клика. |
| `AVERAGE_CRR` | «Оптимизация доли рекламных расходов». |
| `MAX_PROFIT` | «Максимум прибыли». |
| `SERVING_OFF` | **«Показы отключены»** (при установке на `network_strategy` — «Показы в РСЯ отключены»). |
| `NETWORK_DEFAULT` | «Стандартные настройки показов в сетях». |
| `HIGHEST_POSITION` | Ручная с фиксированной максимальной позицией (legacy). |
| `MANUAL_CPM` | Ручное управление ставками CPM (медийная). |
| `WEEKLY_CLICK_PACKAGE` | «Недельный пакет кликов» (legacy). |
| `MAXIMUM_COVERAGE` | «Максимум показов по минимальной цене» (охватная). |

|---|---|

| `ADD_OPENSTAT_TAG` | «Добавлять метку _openstat к ссылкам» |
| `ADD_TO_FAVORITES` | «Добавить в избранное» |
| `REQUIRE_SERVICING` | «Останавливать объявления при недоступности сайта» |
| `EXCLUDE_PAUSED_COMPETING_ADS` | «Не учитывать автоматически остановленные объявления конкурентов» |
| `MAINTAIN_NETWORK_CPC` | «Корректировать ставки в сетях» |
| `DAILY_BUDGET_SPEND_MODE = DISTRIBUTED` | «Распределённый режим дневного бюджета» (равномерный) |
| `DAILY_BUDGET_SPEND_MODE = STANDARD` | «Стандартный режим дневного бюджета» (быстрый расход) |

## Примеры хорошо / плохо

| «Шаг C5 — кластеризация» | «Теперь разобью семантику на группы» |
