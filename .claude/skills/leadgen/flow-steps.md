> Используй якорь (C1, A2, O3.1 и т.д.) чтобы указать Claude какой шаг дорабатывать.
> Пример: «доработай C4» = «доработай шаг сбора семантики в флоу создания».

## ВЕТКА 1: СОЗДАНИЕ (C1–C10)

### C1 — Определение параметров

| | |
|---|---|
| **Якорь** | `C1` |
| **Что происходит** | Спросить у пользователя: город, тематика, тип размещения, бюджет, посадочная |
| **MCP-инструменты** | — |
| **Файлы скилла** | — |
| **Вход** | Запрос пользователя |
| **Выход** | Город, тематика, тип (поиск/РСЯ/ЕПК), бюджет, посадочная (site/LP) |

### C2 — Подготовка конфигурации

| | |
|---|---|
| **Якорь** | `C2` |
| **Что происходит** | Получить конфиг города, конверсии, UTM, конвенцию именования |
| **MCP-инструменты** | `get_city_config`, `get_conversion_values`, `get_utm_config` |
| **Файлы скилла** | `config/campaign_naming.md` |
| **Вход** | Город, тематика, тип размещения |
| **Выход** | client_login, counter_id, priority_goals_json, tracking_params, шаблон имени |

### C3 — Анализ аналогов

| | |
|---|---|
| **Якорь** | `C3` |
| **Что происходит** | Найти аналогичную кампанию в другом городе, получить статистику и объявления, написать анализ. **+** посмотреть топ-5 SERP по ключам тематики (если `tokens.env` настроен) — выжать УТП и CTA конкурентов. |
| **MCP-инструменты** | `get_campaigns`, `get_campaign_stats`, `get_ads` |
| **Скилл `serp-monitor`** | `scripts/search.sh --query "<ключ>" --region <geo> --domain etagi.com` (×3-5 топовых ключей тематики) — опционально, если tokens.env заполнен |
| **Файлы скилла** | — |
| **Вход** | Тематика, тип размещения |
| **Выход** | Секция «Анализ и гипотезы» — цифры, паттерны, идеи для улучшения + список паттернов конкурентов (если был SERP-разбор) |

### C4 — Сбор семантики

| | |
|---|---|
| **Якорь** | `C4` |
| **Что происходит** | Многораундовое расширение: seeds (БЕЗ города) → Wordstat → фильтрация → финализация |
| **MCP-инструменты** | `get_semantic_cluster`, `check_search_volume` (батчи по 10) |
| **Файлы скилла** | `library/selling_modifiers.md`, `mcp/negative_keywords.md` |
| **Вход** | Тематика, город, geo_region_id |
| **Выход** | Массив [{phrase, shows}] 30-80 фраз + мини-кластер «с уточнением города» + ≥150 минус-слов |
| **Раунды** | R1: матрица seeds (БЕЗ города) → R2: Wordstat расширение → R3: второй уровень (опц.) → R4: финализация + валидация (доля без города ≥70%, 4+ типов модификаторов, ≥150 минусов) |

### C5 — Кластеризация

| | |
|---|---|
| **Якорь** | `C5` |
| **Что происходит** | Разбить семантику на 7 интент-кластеров (из selling_modifiers.md) + гео + объект + catch-all. Показать таблицу пользователю |
| **MCP-инструменты** | — |
| **Файлы скилла** | `library/selling_modifiers.md` |
| **Вход** | Массив [{phrase, shows}] из C4 |
| **Выход** | Таблица кластеров (утверждённая пользователем): ≥4 кластеров, название, фразы, заголовок-идея, посадочная |

### C5.5 — Детальное превью для отсмотра спецом

| | |
|---|---|
| **Якорь** | `C5.5` |
| **Что происходит** | Выгружает полный список {ключ, shows} по каждому кластеру + минус-слова в файл `campaigns/<slug>_preview.md`. Итеративный цикл правок со спецом до одобрения. |
| **MCP-инструменты** | — (возможно дополнительно `check_search_volume` если спец попросит добавить ключи) |
| **Файлы скилла** | `templates/preview.md` |
| **Вход** | Утверждённая структура кластеров из C5 |
| **Выход** | `campaigns/<slug>_preview.md` с отметкой «одобрено» |
| **Правило** | БЕЗ явного «ок/продолжай» — НЕ переходим к C6 |

### C6 — Создание контента

| | |
|---|---|
| **Якорь** | `C6` |
| **Что происходит** | Для каждого кластера: заголовки, тексты, посадочная URL, быстрые ссылки, проверка URL, юрпроверка |
| **MCP-инструменты** | `get_sitelink_templates`, `build_landing_url` |
| **Файлы скилла** | `library/titles.md`, `library/texts.md`, `library/extensions.md`, `references/site_structure.md` |
| **Внешние** | `LEGAL.md` (юрпроверка), браузер (проверка URL) |
| **Вход** | Кластеры из C5, город, тематика |
| **Выход** | Готовый контент на кластер: Title, Title2, Text, Href, sitelinks, callouts |

### C7 — Создание через API

| | |
|---|---|
| **Якорь** | `C7` |
| **Что происходит** | Создать кампанию + цикл по кластерам: группа → ключевые → UTM → автотаргетинг → 3 объявления → sitelinks → уточнения |
| **MCP-инструменты** | `add_campaign`, `add_adgroup`, `add_keywords`, `update_adgroup`, `update_autotargeting`, `add_ad` ×3, `add_sitelinks`, `update_ad`, `add_ad_extension` |
| **Файлы скилла** | `library/extensions.md` (уточнения) |
| **Вход** | Контент из C6, priority_goals_json из C2, tracking_params из C2 |
| **Выход** | campaign_id, [adgroup_ids], [banner_ids], [sitelink_set_ids] |
| **Чеклист/группу** | adgroup ✓ → keywords ✓ → UTM ✓ → autotargeting ✓ → 3 ads ✓ → sitelinks ✓ → уточнения ✓ |

### C8 — Кросс-минусация

| | |
|---|---|
| **Якорь** | `C8` |
| **Что происходит** | Построить кросс-минус-слова между группами: маркеры каждого кластера → минус для остальных |
| **MCP-инструменты** | `add_negative_keywords` |
| **Файлы скилла** | — |
| **Вход** | Все ключевые по группам из C7 |
| **Выход** | Минус-слова на уровне групп (не кампании) |

### C9 — Метки

| | |
|---|---|
| **Якорь** | `C9` |
| **Что происходит** | Назначить метки (Лидген, тематика, направление) на все объявления |
| **MCP-инструменты** | `add_labels`, `get_labels` |
| **Файлы скилла** | `config/labels.md` |
| **Вход** | campaign_id, все banner_ids из C7 |
| **Выход** | Метки назначены, проверены |

### C10 — Документирование

| | |
|---|---|
| **Якорь** | `C10` |
| **Что происходит** | Создать файл кампании по шаблону, зафиксировать кластеры, гипотезы, фильтры |
| **MCP-инструменты** | — |
| **Файлы скилла** | `campaigns/template.md` |
| **Вход** | Все данные из C1–C9 |
| **Выход** | Файл `campaigns/<slug>.md` |
