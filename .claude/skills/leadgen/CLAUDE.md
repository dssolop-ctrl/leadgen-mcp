# Skill `leadgen` — Dev Notes

> **Этот файл — для разработчиков скилла**, не для агента-пользователя.
> Runtime-инструкции (как Claude должен запускать кампании, какие пороги, какие правила копирайта) — в `skill.md` и связанных файлах.
> **Загружать этот CLAUDE.md только когда явно работаем над содержимым скилла** (правим `branches/`, `library/`, `references/`, `config/`, `mcp/`, `templates/` или `skill.md`). При обычной dev-сессии MCP-сервера — не загружать.
> При активации скилла слэш-командой `/leadgen` — также **не загружать этот файл**: грузится `skill.md`.

## Двойное дерево — обязательная синхронизация

Скилл живёт параллельно в двух деревьях:
- `.claude/skills/leadgen/` — для Claude Code.
- `.codex/skills/leadgen-codex/` — для OpenAI Codex (роутится через `.codex/mcp.json`).

**Любая правка структуры (`branches/`, `library/`, `references/`, `config/`, `mcp/`, `scripts/`, `templates/`) обязана попасть в обе ветки.**

Перед коммитом — `diff -r .claude/skills/leadgen/ .codex/skills/leadgen-codex/` (исключить локальные `lessons_registry.md`, если разошлись осознанно).

Расхождения только с явной пометкой `Codex-only` / `Claude-only` в коммит-сообщении + запись в `references/lessons_registry.md` соответствующей ветки с указанием причины.

## Frontmatter `skill.md`

`description` — ≤ 50 слов, плотные триггеры. Каждое лишнее слово платится токенами в каждом сообщении, где скилл активен. Не растягивать.

## Правила правки контента скилла

1. **Lazy-load mantra:** добавляя контент, всегда думай — на каком шаге это нужно? Прописывай явно в `skill.md` секции «Правила ленивой загрузки», какой файл читать на каком шаге. Не «грузить на всякий случай».
2. **Якоря шагов** (C1–C10, R1–R11 и т. д.) — стабильный контракт. Не переименовывай, добавляй с суффиксами (`R6.5`, `O3.10b`).
3. **Backlog улучшений** — `docs/skill_improvement_backlog.md`, не в дереве скилла. Грузить только если пользователь явно спрашивает про планы.
4. **Уроки от ошибок API** — `references/lessons_registry.md`. Запись ⇒ один абзац: симптом → причина → правильный паттерн.

## Тесты

Автотестов нет. Регрессии ловятся через end-to-end: создание кампании на тестовом аккаунте + проверка через Claude/Codex в обоих деревьях.

## История

- Внутрискилловые уроки (тонкости конкретных шагов, баги API) → `references/lessons_registry.md`.
- Архитектурные изменения скилла (новые ветви, рефакторинг роутеров, перенос файлов) → `RECENT-CHANGES.md` в корне репо.
