Обнови CLAUDE.md на основе изменений, сделанных в текущем диалоге.

## Алгоритм

### 1. Проанализируй изменения в диалоге
- Какие файлы созданы/изменены/удалены
- Какие новые фичи или модули добавлены
- Какие схемы данных изменены (БД, типы, интерфейсы)
- Какие API endpoints добавлены/изменены
- Какие зависимости добавлены

### 2. Прочитай текущий CLAUDE.md
- Определи структуру и стиль документации
- Найди секции, которые затрагивают изменения
- Отметь что может быть устаревшим

### 3. Обнови соответствующие секции
- **Структура проекта** — новые файлы/папки
- **Архитектура/схемы** — новые сущности, поля, связи
- **API/endpoints** — новые роуты, параметры
- **Конфигурация** — новые env переменные
- **Команды** — новые скрипты, CLI команды
- **Примеры кода** — если изменились API

### 4. Проверь на устаревшее
- Удали упоминания удалённых файлов/функций
- Исправь неактуальные примеры
- Обнови enum/статусы если добавлены значения

## Формат документирования

При добавлении новой фичи:
```
### Название фичи

Краткое описание.

**Flow:** (ASCII-диаграмма если сложная логика)

**Ключевые файлы:**
| Файл | Роль |
|------|------|
| path/to/file.ts | Описание |

**Примеры:** (если есть API)
```

## Принципы

- Сохраняй существующий стиль документации
- Не дублируй информацию
- Документируй "почему", а не только "что"
- Держи примеры кода минимальными и актуальными
