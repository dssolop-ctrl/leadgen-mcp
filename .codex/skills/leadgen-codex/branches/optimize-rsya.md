## ВЕТКА 3-R: ОПТИМИЗАЦИЯ РСЯ (O-RSYA)

> **Когда использовать:** оптимизация конкретной РСЯ-кампании (не поисковой).
> Подветки O0 (аудит сущностей) и O1 (креативный анализ) — общие, применяются как есть.
> Ниже — РСЯ-специфичные подветки.

<!-- OR.1 — чистка площадок -->
### OR.1. Чистка площадок-дренажей

**Когда:** CPA выше порога ИЛИ через 3 дня после запуска (обязательно).

```
1. get_custom_report(
     report_type="CUSTOM_REPORT",
     field_names="CampaignName,Placement,Impressions,Clicks,Cost,Conversions",
     date_from=<от запуска или 14 дней назад>,
     date_to=<сегодня-1>,
     filter_json='[{"Field":"CampaignId","Operator":"EQUALS","Values":["<id>"]}]',
     goal_ids=<form_goal,call_goal>,
     attribution="LYDC"
   )
2. Отфильтровать площадки:
   - Расход > 500 ₽ И Conversions = 0  → кандидат на блок
   - Расход > 200 ₽ И CTR < 0.05%      → кандидат на блок
3. Показать пользователю список (в review) или сразу блокировать (в auto):
   add_blocked_placements(campaign_id, places=<список>)
4. Фиксация в campaigns/<slug>.md раздел «Заблокированные площадки»
```

**Каденция:** через 3 дня после запуска, далее раз в 2 недели.

<!-- OR.2 — A/B картинок -->
### OR.2. A/B тест картинок

**Когда:** CTR ниже нормы (< 0.3%) ИЛИ накопилось > 10 000 показов без роста конверсий.

```
1. get_ad_stats (по объявлениям кампании) → найти объявления с худшим CTR
2. Проанализировать их картинки (get_ad_images с AdImageHash) — визуально
3. Сгенерировать 2-4 новых варианта картинки (другие V1-V7 или другая сцена)
   - НЕ превышать общий лимит 20 генераций (учитывать уже созданные)
4. add_ad_image + update_ad (ad_image_hash=<новый>) — заменить картинку в худших объявлениях
   ИЛИ добавить новые объявления (add_ad ×3 с новыми картинками), старые приостановить (manage_ads action=suspend)
5. Мониторить 7 дней, сравнить CTR старых vs новых
```

<!-- OR.3 — расширение аудиторных таргетов -->
### OR.3. Расширение аудиторных таргетов (LAL / Ретаргет / Интересы)

**Когда:** накопилось достаточно данных (> 5 000 визитов, > 20 конверсий).

```
1. metrika_get_audience(counter_id) → сегменты Метрики
2. Создать LAL-сегмент на 3-5% по конвертерам (через интерфейс Метрики; API пока не покрывает)
3. add_audience_targets(adgroup_id, audience_id=<LAL_segment>) → новая группа или расширение существующей
4. Ретаргет: retargeting_list на сегмент «был на сайте, не оставил заявку»
```

<!-- OR.4 — частотное ограничение -->
### OR.4. Частотное ограничение (frequency cap)

**Когда:** много показов одному пользователю без конверсии (видно в reach_frequency_stats).

```
1. get_reach_frequency_stats(campaign_id) → распределение частоты
2. Если > 20% показов приходится на частоту 5+ без конверсий — включить cap
3. update_campaign(settings=[{"option":"FREQUENCY_CAP","value":"DAY_3"}])
   (формат API уточняется; пока — через интерфейс)
```

<!-- OR.5 — корректировки по устройствам/полу/возрасту -->
### OR.5. Корректировки bid_modifiers

**Когда:** явные перекосы в статистике по устройству/полу/возрасту (> 20% разницы CPA).

```
1. get_custom_report с dimension Device,Gender,Age → сравнить CPA
2. set_bid_modifiers на адгруппу или кампанию:
   - desktop vs mobile (обычно mobile дороже — корректировка -20%)
   - age 25-34 vs 45+ (часто 45+ дешевле в регионах — поднять ставку +15%)
3. Мониторить 7 дней
```
