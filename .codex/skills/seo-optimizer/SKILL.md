# SEO Optimizer

Выполняй SEO-аудит и улучшай страницу по чеклисту ниже.

## Workflow

1. Проанализируй страницу:
- Вытащи текущие `title` и `description` из metadata/generateMetadata.
- Проверь структуру `H1/H2/H3`.
- Определи основные ключевые слова по содержимому.
- Проверь внутренние ссылки.
- Проверь `Schema.org`, `Open Graph`, `canonical`.

2. Оцени страницу по чеклисту (18 баллов):

### Title Tag (0-4)
- 50-60 символов.
- Ключевое слово в начале.
- Уникальность и кликбейт без спама.
- Бренд в конце (опционально).

### Meta Description (0-4)
- 150-160 символов.
- Начало с глагола действия.
- Ключевое слово встроено естественно.
- Есть призыв к действию.

### Контент (0-6)
- Один `H1` с ключевым словом.
- Ключевое слово в первых 100 словах.
- Минимум 300 слов контента.
- Логичная структура `H2/H3`.
- Изображения с осмысленным `alt`.
- 3-5 релевантных внутренних ссылок.

### Технический SEO (0-4)
- Корректный `canonical`.
- `Open Graph` (`og:title`, `og:description`, `og:image`).
- `Schema.org` разметка.
- Читаемый URL/slug с ключевым словом.

3. Сформируй отчет:

```markdown
## SEO-аудит: [Название страницы]

Текущий балл: XX/18
Статус: Отлично (15+) | Нужны улучшения (10-14) | Критично (<10)

### Текущее состояние
- Title: "[...]" (N символов)
- Description: "[...]" (N символов)
- H1: "[...]"
- URL: /...

### Рекомендации
#### Критичные
1. ...
   - Текущее: ...
   - Рекомендуемое: ...

#### Важные
1. ...

#### Опционально
1. ...
```

4. После одобрения пользователя внеси изменения:
- Обнови metadata (`title`, `description`, `openGraph`).
- Добавь/обнови `Schema.org` JSON-LD.
- Улучши контент: ключевая фраза в начале, корректная структура заголовков, внутренние ссылки.

5. Перед завершением пройди финальный чеклист:
- `title` 50-60 символов и с ключевым словом.
- `description` 150-160 символов и с CTA.
- Один `H1`.
- Ключевое слово в первом абзаце.
- Достаточный объем текста (300+ слов, если формат страницы это предполагает).
- Проставлены `Open Graph`, `canonical`, `Schema.org`.

## Шаблоны (настройте под свой проект)

### Страница продукта/услуги
- Title: `[Продукт/услуга] - [УТП] | Название сайта`
- Description: `[Действие] [продукт/услуга]. [Преимущество]. [CTA].`

### Страница категории
- Title: `[Категория] - [описание] | Название сайта`
- Description: `[Действие] лучшие [категория]. [Количество] вариантов. [CTA].`

## Schema.org базовые заготовки

### WebPage
```json
{
  "@context": "https://schema.org",
  "@type": "WebPage",
  "name": "Название страницы",
  "description": "Описание страницы"
}
```

### BreadcrumbList
```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": []
}
```

## Featured Snippets

Используй один из форматов:
- Короткое определение 40-60 слов с ключом в начале.
- Нумерованный список шагов.
- Таблица с интерпретацией/сравнением.
