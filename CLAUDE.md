# Direct MCP & Leadgen Skill — DEV MODE (по умолчанию)

> **Этот проект — разработка скилла `leadgen` и MCP-сервера `yandex-direct` (плюс смежные скиллы `vk-ads`, `yandex-metrika`, `demand-research`, `serp-monitor`, `seo`).**
> По умолчанию Claude работает в **dev-режиме**: правит Go-код MCP, контент скиллов, конфиги, документацию. **Скилл-правила (как создавать кампании, какие пороги CPA, юр. проверка контента и т. п.) НЕ применяются автоматически.**

## Когда применяются скилл-правила (RUNTIME-режим)

Только при **явной активации скилла** пользователем через слэш-команду:

| Слэш-команда | Скилл |
|---|---|
| `/leadgen` | leadgen — создание / анализ / оптимизация кампаний недвижимости |
| `/yandex-direct` | yandex-direct — узкие операции с Яндекс Директом |
| `/yandex-metrika` | yandex-metrika — аналитика и отчёты |
| `/vk-ads` | vk-ads — реклама VK |
| `/demand-research` | исследование спроса, упущенные ключевые |
| `/serp-monitor` | мониторинг позиций |
| `/seo` | SEO-оптимизация страниц |

В runtime-режиме грузятся `.claude/skills/<name>/skill.md` и связанные `branches/`, `library/`, `references/` — но **только по правилам ленивой загрузки самого скилла**. **Без слэш-команды эти файлы и связанные runtime-документы (`METRIKA-ADS-RULES.md`, `VK-ADS-RULES.md`, `LEGAL.md`, `PROJECTS.md`, `agent-direct_wordstat.md`, `agent-vk.md`) в контекст не загружать.**

Если пользователь устно просит выполнить рекламную задачу без слэш-команды — уточнить, нужен ли runtime-режим, и не подгружать скилл «на всякий случай».

## Что лежит где

| Путь | Назначение | Грузить при старте? |
|---|---|---|
| `RECENT-CHANGES.md` | **Журнал изменений проекта.** Постоянный, переносит контекст между ПК и сессиями Claude/Codex. | **Да — последние 1–2 пункта.** |
| `CLAUDE.md` (этот файл) | Dev-режим по умолчанию, маршрутизация. | Да (автоматически). |
| `server/CLAUDE.md` | Dev-инструкции MCP-сервера: build, структура пакетов, как добавить инструмент. | Только если cwd в `server/` или меняем код MCP. |
| `.claude/skills/leadgen/CLAUDE.md` | Dev-инструкции скилла leadgen: правило двойного дерева, lazy-load, sync. | Только если cwd в `.claude/skills/leadgen/` или редактируем сам скилл. **НЕ грузить при обычной dev-сессии.** |
| `TECHNICAL.md` | Технические детали (runtime архитектура, API quirks, известные сбои, отладка). | **Нет — лениво, только при сбое или поиске специфики.** |
| `server/` | Go-исходники MCP-сервера. |
| `.claude/skills/<name>/` | Скиллы Claude Code. Содержимое не читать без активации скилла или dev-задачи по нему. |
| `.codex/skills/leadgen-codex/` | Зеркало скилла leadgen для Codex. |
| `campaigns/` | Журналы кампаний (markdown). Runtime-данные. |
| `docs/` | Заметки исследований, превью креативов, диаграммы. |
| `MEMORY.md` (в `~/.claude/projects/...`) | Auto-memory Claude Code (вне репо). |

## Старт сессии — короткий чеклист

1. Прочитать **последние 1–2 пункта `RECENT-CHANGES.md`** — узнать состояние и недавние решения.
2. По задаче пользователя определить режим:
   - **Dev MCP** → грузить `server/CLAUDE.md` при первом обращении к `server/`.
   - **Dev скилла** → грузить `.claude/skills/leadgen/CLAUDE.md` при первом обращении к скилл-файлам.
   - **Runtime скилла (слэш-команда)** → грузится содержимое скилла, не этот CLAUDE.md.
3. **Не грузить превентивно:** `TECHNICAL.md`, `METRIKA-ADS-RULES.md`, `VK-ADS-RULES.md`, `LEGAL.md`, `PROJECTS.md`, `agent-*.md`, `usecases.md` (если когда-либо вернётся), runtime-`skill.md` любых скиллов.

## После значимых изменений — обновить журнал

Если в сессии:
- меняли код MCP / собирали контейнер / правили Dockerfile;
- редактировали структуру скилла, ветви, библиотеки;
- проводили крупную чистку, миграцию, рефакторинг документации;
- принимали архитектурное решение, которое влияет на следующие сессии,

→ дописать новый пункт **сверху** в `RECENT-CHANGES.md`. Формат — как у предыдущих: дата, контекст, W1/W2/... (волны), эффект.

Тривиальные правки (опечатки, мелкие фиксы) в журнал не идут.

## Двойное дерево скиллов — обязательная синхронизация

`.claude/skills/leadgen/` ↔ `.codex/skills/leadgen-codex/` — параллельные структуры (`branches/`, `library/`, `references/`, `config/`, `mcp/`, `templates/`). **Любая правка флоу/референсов/библиотек/конфигов синхронизируется в обе ветки.** Перед коммитом — `diff -r` между ними.

Расхождения только с явной пометкой `Codex-only` / `Claude-only` в коммите + запись причины в `references/lessons_registry.md` соответствующей ветки.
