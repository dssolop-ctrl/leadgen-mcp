# Яндекс Директ (MCP)

Используй этот навык когда задача связана с рекламными кампаниями Яндекс Директа:
- Просмотр и управление кампаниями, группами, объявлениями
- Ставки и корректировки ставок
- Ключевые фразы и автотаргетинг
- Статистика и отчёты
- Быстрые ссылки, изображения, видео, визитки
- Фиды, смарт-баннеры, динамические объявления
- Управление клиентами агентства

## Подключение к MCP-серверу

Сервер доступен по HTTP — подключение напрямую без промежуточных утилит:

- URL: `https://direct-mcp.aatex.ru/mcp`
- Заголовок: `Authorization: Bearer $YANDEX_DIRECT_TOKEN`

Токен берётся из переменной окружения `YANDEX_DIRECT_TOKEN`, заданной в конфиге OpenClaw.

Если клиент не поддерживает HTTP MCP напрямую, используйте мост:

```bash
npx -y mcp-remote https://direct-mcp.aatex.ru/mcp --header "Authorization: Bearer $YANDEX_DIRECT_TOKEN"
```

## Типовые запросы пользователей

- «Покажи мои кампании» → get_campaigns с field_names: ["Id", "Name", "State", "Status"]
- «Какая статистика у кампании 123?» → get_campaign_stats с campaign_id
- «Покажи объявления в группе 456» → get_ads с adgroup_ids
- «Какие ключевые фразы в кампании?» → get_keywords с campaign_ids
- «Сколько потратил за неделю?» → get_account_stats с датами
- «Останови кампанию 123» → suspend_campaigns с ids
- «Измени ставку на фразе» → set_keyword_bids
- «Покажи поисковые запросы» → get_search_queries с campaign_id
- «Добавь минус-фразу» → update_campaign с NegativeKeywords
- «Создай новую кампанию» → add_campaign
- «Какие клиенты у агентства?» → get_agency_clients (для агентств)

## Правила безопасности

- Никогда не выводи токен в ответах
- Никогда не записывай токен в файлы, git или логи
- При ошибке авторизации сообщи пользователю и попроси обновить токен в конфиге
