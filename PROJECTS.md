## Продукт

Этажи — федеральная сеть услуг недвижимости. `leadgen-mcp` управляет performance-рекламой для лидогенерации по городским сайтам Этажей: вторичка, новостройки, загородная, аренда, коммерческая, ипотека, агентские, HR.

Основной паттерн доменов: `{city}.etagi.com` (исключения и tier-разметка живут в `get_city_config`). Лендинги — `*.etagi.net`. Этажи Прайм — `etagiprime.com`.

## Аккаунты

- **Директ** — агентство `etagi click` (default). Для большинства вызовов `account` не указывать.
- **Метрика** — всегда `account="etagi.agent"`. Без явного account аналитика возвращает не тот контекст.
- **VK Ads** — default из `server/config.yaml`.

## Цели конверсии — правила выбора

Стратегия Директа использует **две цели одновременно через `priority_goals`**, не одну микроцель:

1. **Цель Общая (все лиды)** — все формы заявок (условие `form_sum_leads`).
2. **Цель Коллтрекинг, реальный звонок** — подтверждённые звонки (условие `received_real_calls`). Если на счётчике коллтрекинга нет — fallback на **«Все звонки»** (условие `all_calls`).

Конкретные ID — через `get_conversion_values(...)`. Названия в обсуждении писать по-человечески: «цель Отправка заявки (123456789)», не голым ID.

**НЕ использовать для оптимизации стратегии:** микроцели (клики, скроллы, просмотры страниц, клики по телефону).

Формы и звонки **не пересекаются** — в отчётах смотреть обе цели.

## CPA-интерпретация

Целевой CPA приходит из `get_conversion_values` для конкретной (city × theme × goal). Поверх него — единая интерпретация:

| Бакет | Условие | Действие |
|---|---|---|
| 🟢 GOOD | CPA ≤ target | Продолжать, готовить масштабирование после накопления данных |
| 🟡 WATCH | target < CPA ≤ 1.5 × target | Чистка поисковых запросов, минусы, CTR групп, A/B текстов |
| 🔴 CRITICAL | CPA > 1.5 × target | Разбор слабых групп, посадочных, запросов, стратегии; возможна пауза |

Снижение бюджета / стратегии — допустимо в любой момент при CRITICAL. Рост >30% за раз — только с явного подтверждения пользователя, желательно в чт-пт (выходные на стабилизацию).

Стратегии: старт `WB_MAXIMUM_CONVERSION_RATE`. Переход на `AVERAGE_CPA` — после стабильных 10+ конверсий/нед в течение 2–3 недель.

## Бюджеты по умолчанию

Канонический источник — MCP-инструмент `get_default_budgets(channel, tier, theme?, target_cpa?)`. Возвращает test/start/scale_min для (channel × tier) с поправкой на тематику и опциональным расчётом от целевого CPA. Поддерживаемые каналы: `search`, `rsya`, `vk`. Tier города — из `get_city_config(city).tier`.

Формула при расчёте от target CPA: `weekly_budget = target_cpa × 10 × 1.2`, округление до 500 ₽. Минимальные пороги (ниже которых автостратегии не обучаются): 5000 ₽ search, 3000 ₽ rsya, 2000 ₽ vk.

## Именование кампаний и файлов

Имя кампании в Директе:

```
<Город> | <Канал> | <Тематика> | <Сегмент> | [site|lp]
```

Примеры: `Омск | Поиск | Вторичка | Однокомнатные | [site]`, `Казань | РСЯ | Новостройки | Общая | [site]`.

Файл истории кампании: `campaigns/<city>_<channel>_<theme>[_<segment>].md`.

В шапке файла кампании обязательно: имя + ID, UTM, счётчик (название + ID), цели (названия + ID), бюджет, стратегия, client_login, регион, домен, метки, текущий state.

После каждой проверки/правки кампании — обновлять файл (история и «Где сейчас»). Задачи в файле кампании НЕ хранятся.

## Минусация

**НЕ минусовать автоматически:**
- Целевые коммерческие запросы тематики: «купить квартиру», «однокомнатная», «вторичка» и т. п. (в рамках своей тематики).
- Районы целевого города.
- «От собственника» в покупке квартир — отдельный коммерческий кластер.
- «Недорого», «дешево» — оставлять, кросс-минусовать между ценовыми группами.
- Ипотечные формулировки в кампаниях покупки, если объявление и посадочная поддерживают сценарий.

**Минусовать:**
- Чужие города и регионы (если кампания не федеральная).
- Информационный интент: «что такое», «как оформить самому», «образец», «скачать», «реферат», «википедия».
- Несоответствующая тематика (аренда в покупке, новостройки во вторичке и наоборот).
- Нецелевые роли: подрядчики, работа, обучение, бесплатно, форум, отзывы — без отдельной гипотезы.
- Запросы с расходом и нулевыми конверсиями после проверки `get_search_queries` с `attribution: LYDC`.

Базовые наборы минус-слов по тематикам — через MCP `get_negative_keyword_guidance(theme, city?, placement?)`. Возвращает блоки (чужие города РФ с авто-исключением собственного, информационные, бесплатно/обучение, нерелевантные услуги, работа, юр., медицина, негатив, тематика, конкуренты), готовую строку через запятую и хард-гейт ≥150 слов. Полный текстовый референс — `.claude/skills/leadgen/mcp/negative_keywords.md` (для ручного просмотра).

## Прочее

- При создании кампании после `add_campaign` НЕ вызывать `resume_campaign` — кампания остаётся в DRAFT/SUSPENDED, активирует пользователь.
- UTM — через `tracking_params` на уровне группы (не на уровне кампании).
- Поиск: `network_strategy="SERVING_OFF"`. РСЯ: Network-стратегия с вложенной структурой (см. `TECHNICAL.md`).
- GEO-кампании — отключать `ENABLE_AREA_OF_INTEREST_TARGETING`.
- Атрибуция отчётов — всегда `LYDC` в `get_campaign_stats` и `get_search_queries`.
- Автотаргетинг — только целевые + узкие, остальные выключены.
- Бюджет в API — недельный, в рублях (число), без микроюнитов.

Полные API-нюансы — `METRIKA-ADS-RULES.md` и `TECHNICAL.md`. Этот файл — только policy «над ними».
